# Issue 174 Pi 兼容性实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 Agent Guard（代理守卫）和 PR Flow（拉取请求流程）在 Pi（编码助手）中可用，同时保持 Codex（编码助手）和 Claude（克劳德）行为不变。

**Architecture:** Agent Guard（代理守卫）使用 Pi（编码助手）原生 `session_start`（会话启动）与 `tool_call`（工具调用）事件逐次调用既有 Python（派森）Hook Router（钩子路由器）。PR Flow（拉取请求流程）注册 Pi（编码助手）专属工具，从 extension（扩展）自身位置运行包内脚本；Python（派森）恢复命令从 `__file__`（当前脚本文件）生成绝对路径。

**Tech Stack:** TypeScript（类型脚本）、Pi Extension API（Pi 扩展接口）、Python（派森）标准库、pytest（派森测试框架）。

## Global Constraints

- Agent Guard（代理守卫）不得导入、配置、调用或修改其他 Pi（编码助手）插件。
- Agent Guard（代理守卫）不得改变 Guard Profile（守卫画像）、Session Focus（会话焦点）、Runtime API（运行时接口）或审计契约。
- Pi（编码助手）适配必须覆盖全部工具调用；Router（路由器）失败时阻断当前调用，不能静默放行。
- PR Flow（拉取请求流程）的脚本路径必须来自安装包自身，`--project .`（项目参数）始终表示目标仓库。
- 不修改 Codex（编码助手）/Claude（克劳德）manifest（清单）、`hooks/hooks.json`（钩子配置）或目标仓库 Pi（编码助手）配置。
- 每项实现先写失败测试；所有提交说明使用简体中文。

- [x] **Step 3: 实现无状态 Router（路由器）桥接**

在 `pi-agent-guard.ts` 中使用 `fileURLToPath(new URL("..", import.meta.url))`（模块位置解析）得到插件根目录，使用 Node（节点）`spawn`（启动子进程）向 `python <plugin>/scripts/hook_router.py` 写入 JSON（数据）。

实现以下内部边界：

```ts
type RouterResult = { status: "allow" | "deny" | "ask" | "observed" | "error"; reason?: string; suggestion?: string };
async function invokeRouter(event: "SessionStart" | "PreToolUse", payload: Record<string, unknown>, cwd: string): Promise<RouterResult>;
```

`session_start`（会话启动）发送 `{session_id, cwd}`。每个 `tool_call`（工具调用）发送完整 `{session_id, cwd, tool_name: event.toolName, tool_input: event.input}`；`allow`（允许）不返回值，`deny`（拒绝）或 `ask`（询问）返回 `{block: true, reason}`。子进程非零、无 JSON（数据）或未知状态返回 `{block: true, reason: "agent_guard_router_failed: ..."}`。不得保存可持久化会话状态、不得调用 UI（界面）确认、不得加载其他插件；只允许在 `session_start`（会话启动）中设置当前进程的 Agent Guard（代理守卫）会话环境变量，供既有技能辅助脚本使用。
