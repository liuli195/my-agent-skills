## 原因

同名源分支复用时，已合并或已关闭的 PR（拉取请求）仍会被识别为当前活动 PR。流程因此使用旧源提交校验最新基线，并错误停止为 `base_outdated`（基线过期），阻断后续批次发布。

## 改动内容

- 仅将 `OPEN`（未合并）状态的 PR（拉取请求）视为活动 PR。
- 已合并和已关闭的同名 PR（拉取请求）不再参与当前批次的基线校验；`complete`（收尾）与 `tweak`（小改）改用当前 `HEAD`（当前提交）并创建新的 PR（拉取请求）。
- 为终态 PR（拉取请求）和活动 PR（拉取请求）增加回归覆盖。

## 能力

### 新增能力

- 无。

### 修改能力

- `pr-flow-plugin`：明确终态同名 PR（拉取请求）不能作为当前 PR（拉取请求）生命周期的活动对象。

## 影响

- 代码：`plugins/pr-flow/skills/pr-flow/scripts/pr_flow.py`。
- 测试：`tests/test_pr_flow_cli.py`。
- 不新增接口、配置、依赖或数据迁移。
