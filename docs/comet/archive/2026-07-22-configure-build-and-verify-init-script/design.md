## Context

量化研究实验室以 `scripts/setup-worktree.ps1` 为工作树环境入口：它在仓库根目录创建或复用 `.venv`（本地虚拟环境），升级 pip（包安装工具）并安装声明依赖。本仓库同样使用 Python（Python 语言）3.12 与 `requirements-dev.txt`（开发依赖清单），但当前没有对应入口，历史记录也显示工作树可能退回系统解释器。

## Goals / Non-Goals

**Goals:**

- 为每个工作树提供一次可重复运行的 PowerShell（命令行脚本）环境初始化入口。
- 创建缺失的 Python（Python 语言）3.12 `.venv`（本地虚拟环境），并安装本仓库声明的开发依赖。
- 让失败立即返回非零退出码，避免后续步骤在不完整环境中继续。

**Non-Goals:**

- 不修改 `requirements-dev.txt`（开发依赖清单）、Build and Verify（构建与验证）配置或持续集成。
- 不运行构建检查、验证、发布或任何仓库业务命令。
- 不修改量化研究实验室。

## Decisions

1. 新增 `scripts/setup-worktree.ps1` 作为唯一入口。
   - 脚本从自身路径计算仓库根目录，因此可从任意 PowerShell（命令行）当前目录调用。
   - 采用量化研究实验室的最小控制流，而非引入新的初始化框架。

2. 缺失环境时使用 `py -3.12 -m venv .venv`（创建虚拟环境）。
   - 持续集成已固定 Python（Python 语言）3.12；复用这一版本避免本地与持续集成漂移。
   - 已存在 `.venv`（本地虚拟环境）时复用其中解释器，不重建或删除用户环境。

3. 只用 `.venv\Scripts\python.exe`（本地解释器）升级 pip（包安装工具）并安装 `requirements-dev.txt`（开发依赖清单）。
   - 本仓库没有生产依赖清单，开发依赖已覆盖测试和验证所需工具。
   - 每个外部命令后检查退出码；失败时立即停止并保留原始错误。

4. 端到端验证仅在用户明确授权安装依赖后执行。
   - 脚本的用户入口会创建环境并访问包源；仓库规则禁止在没有明确授权时安装依赖。
   - 在未获授权前，测试仅覆盖脚本结构与失败传播，不能将其替代为已完成的端到端证据。

## Risks / Trade-offs

- [本机没有 Python（Python 语言）3.12 启动器] → 脚本直接返回启动器错误，不回退到系统默认 Python（Python 语言）。
- [依赖安装受网络或包源影响] → 保留命令错误并允许用户授权后重试，不静默跳过。
- [已有 `.venv`（本地虚拟环境）版本不一致] → 为保护现有工作树，脚本复用环境；用户可显式删除后重新运行。

## Migration Plan

新增脚本不改变已有自动化。回滚时删除该脚本及其对应测试和规格即可。
