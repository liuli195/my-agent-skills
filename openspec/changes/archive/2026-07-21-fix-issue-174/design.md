## Context

Agent Guard（代理守卫）当前通过 Codex（编码助手）和 Claude（克劳德）生命周期 Hook（钩子）调用 Python（派森）Hook Router（钩子路由器），并且运行时只接受这两个 source（来源）。Pi（编码助手）以 package（包）发现扩展和技能，不加载这套 Hook（钩子）清单，但提供可阻断的原生会话和工具调用事件。

PR Flow（拉取请求流程）在插件缓存中包含完整脚本，但技能示例和 `nextCommand`（下一命令）硬编码源码仓库路径。Codex（编码助手）和 Claude（克劳德）目前依赖代理自行改用缓存绝对路径；Pi（编码助手）从本地 package（包）加载时同样无法可靠执行该相对路径。

本变更保持单一 OpenSpec（开放规格）change（变更）：两项适配共享 Pi（编码助手）运行时边界和目标仓库端到端验证。它们不共享新的业务能力、配置格式或发布机制。

## Goals / Non-Goals

**Goals:**

- 在不改变现有 Guard Runtime（守卫运行时）语义的前提下，让 Pi（编码助手）自动把会话启动和工具调用送入既有 Agent Guard（代理守卫）Router（路由器）。
- 让 Pi（编码助手）使用已安装 PR Flow Plugin（拉取请求流程插件）自身的脚本，并使恢复命令在任何目标仓库可执行。
- 保持 Codex（编码助手）和 Claude（克劳德）的清单、Hook（钩子）和已存在调用路径兼容。

**Non-Goals:**

- 不修改 Guard Profile（守卫画像）结构、Session Focus（会话焦点）生命周期、权限规则或 Runtime API（运行时接口）。
- 不依赖、导入、配置、调用或修改其他 Pi（编码助手）插件；不增加 Agent Guard（代理守卫）的新权限模型、目标仓库 `.pi`（Pi 配置）、全局记忆或子代理框架。
- 不改变 PR Flow（拉取请求流程）的 Git（版本控制）、GitHub（代码托管平台）、验证或发布规则。

## Decisions

### Pi 适配仅作为现有运行时的宿主桥接

在 Agent Guard Plugin（代理守卫插件）内增加 Pi（编码助手）专属 extension（扩展）。它从自身安装位置定位现有 Python（派森）Router（路由器），并把 Pi（编码助手）会话与工具输入标准化为既有 lifecycle envelope（生命周期信封），新增 `source=pi`（来源）但不新增 Guard Profile（守卫画像）字段或评估分支。

适配层直接使用 Pi（编码助手）原生会话和工具调用事件。Guard Router（守卫路由器）继续产生既有 allow（允许）或 deny（拒绝）结果；适配层只进行 Pi（编码助手）输入、会话标识和结果格式转换，并把每次工具调用只交给该 Router（路由器）判定一次。

替代方案：复制或重写 Guard Runtime（守卫运行时）会造成宿主语义漂移；依赖其他 Pi（编码助手）插件的跨扩展接口会耦合独立领域；两者均不采用。

### PR Flow 由 Pi 专属执行入口定位脚本

在 PR Flow Plugin（拉取请求流程插件）内增加 Pi（编码助手）专属 extension（扩展）入口。入口从 extension（扩展）模块自身位置解析包内 `pr_flow.py`（PR Flow 脚本），并保留当前 Pi（编码助手）项目目录作为 `--project .`（项目参数）的语义。

共享技能说明只增加 Pi（编码助手）可用时使用该入口的分流，不替换 Codex（编码助手）和 Claude（克劳德）的已安装脚本调用方式。Python（派森）脚本生成恢复命令时以 `__file__`（当前脚本文件）而非源码仓库相对路径定位自身。

替代方案：依赖技能文本相对路径、硬编码用户本机路径或期待宿主自动切换 CWD（当前工作目录）均不可靠，因此不采用。

### 回归验证覆盖宿主边界

测试分别覆盖 Pi（编码助手）适配输入与既有 Router（路由器）输出、Pi（编码助手）PR Flow（拉取请求流程）入口在外部临时仓库执行、恢复命令脚本位置以及 Codex（编码助手）/Claude（克劳德）既有 package（包）清单与 Hook（钩子）不变。最终端到端验证在两个目标仓库中从用户技能入口运行。

## Risks / Trade-offs

- [Pi（编码助手）原生生命周期事件与当前版本不一致] → 在实现前用本机类型定义和最小端到端工具调用验证事件输入、阻断结果和会话标识。
- [Pi（编码助手）工具输入无法完整映射为命令] → 保持既有 Router（路由器）的安全忽略行为；只对可标准化输入执行命令守卫，不新增推测性解析。
- [绝对恢复命令的路径含空格] → 使用当前 Python（派森）解释器和平台安全引用生成命令，并增加 Windows（视窗）路径回归测试。
- [Pi（编码助手）适配被 Codex（编码助手）或 Claude（克劳德）误加载] → Pi（编码助手）资源仅置于其 convention（约定）目录；现有宿主 manifest（清单）不声明该资源，并以 package（包）测试断言隔离。

## Migration Plan

1. 随插件源码发布 Pi（编码助手）extension（扩展）；既有 Guard Profile（守卫画像）和 PR Flow（拉取请求流程）目标仓库配置无需迁移。
2. 在当前 Pi（编码助手）环境验证自动生命周期和 PR Flow（拉取请求流程）入口。
3. 若 Pi（编码助手）适配出现问题，移除或禁用 Pi（编码助手）extension（扩展）即可恢复现状；Codex（编码助手）和 Claude（克劳德）不受影响。
