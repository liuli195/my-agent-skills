## Root Cause（根因）

最终写入确认仍使用旧 yes/no（是/否）模板，只能表达“写入”或“停止”，没有把 GitHub（代码托管平台）远端待办执行顺序纳入用户选择。

## Fix（修复）

在 `questionnaire.md`（问答模板）的最终写入确认场景中，把固定选项替换为：

- 不写入，放弃本次配置。
- 只写入本地配置。
- 按 remote tasks（远端待办）完成 GitHub（代码托管平台）配置，然后再写入本地配置。

同时补充边界：GitHub（代码托管平台）配置由 agent（代理）执行，插件不提供 GitHub（代码托管平台）配置脚本能力。

## Non-goals（非目标）

- 不新增 GitHub API（接口）写入脚本。
- 不改变 `pr_flow.py init`（初始化脚本）本地写入语义。
- 不展示完整 YAML（配置格式）。
